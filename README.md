# 双人健身进度看板

一个专为情侣/伙伴设计的健身减肥进度可视化看板，支持双人数据同步、日历热图、趋势分析和 GitHub 云端存储。

## ✨ 功能特性

### 📊 双人数据分区
- 左侧：用户数据
- 右侧：女朋友数据
- 完全对称设计，数据独立

### 📅 智能日历
- 月视图显示每日体重记录
- 增长标红色 🔴，降低标绿色 🟢
- 点击日期查看详情
- 支持查看历史月份

### 📈 数据可视化
- 体重变化趋势图
- 目标进度显示
- 连续打卡天数
- 日环比、总变化计算

### 🔄 云端同步
- 数据加密存储到 GitHub Gist
- 多设备实时同步
- 端到端加密保护隐私

### 📱 移动友好
- 响应式设计，手机电脑都能用
- 快速输入今日体重
- 一键填充昨日数据

## 🚀 快速开始

### 1. 获取 GitHub Token
1. 登录 [GitHub](https://github.com)
2. 点击右上角头像 → **Settings**
3. 左侧菜单 **Developer settings** → **Personal access tokens** → **Tokens (classic)**
4. 点击 **Generate new token** → **Generate new token (classic)**
5. 填写：
   - **Note**: `Fitness Tracker`
   - **Expiration**: 选择 `No expiration`
   - **Select scopes**: **只勾选 `gist`**
6. 点击 **Generate token**
7. **立即复制 Token** 并保存到安全地方

### 2. 打开应用
1. 双击 `index.html` 在浏览器中打开
2. 或通过本地服务器访问：
   ```bash
   # Python
   python3 -m http.server 8000
   
   # 然后在浏览器访问 http://localhost:8000
   ```

### 3. 首次设置
1. 点击右上角 **设置** 按钮
2. 输入 GitHub Token
3. 设置加密密码（用于保护数据隐私）
4. 设置两人名称和目标体重
5. 点击 **保存设置**

## 📖 使用指南

### 每日记录体重
1. 在对应区域的输入框中输入今日体重
2. 点击 **提交** 或按 Enter 键
3. 系统自动计算变化并更新所有图表

### 快速操作
- **快速填充昨日**：点击按钮自动填充昨日体重
- **修改历史数据**：点击日历中的日期查看详情
- **补填遗漏数据**：直接输入过去日期的体重

### 数据同步
- **手动同步**：点击顶部 **同步数据** 按钮
- **自动同步**：每次记录后自动同步到云端
- **数据恢复**：新设备打开应用会自动拉取云端数据

## 🔍 常见问题

### Q: 为什么需要 GitHub Token？
A: 用于创建和更新你的私有 Gist 来存储数据。Token 只用于 GitHub API，我们无法访问你的其他 GitHub 数据。

### Q: 数据真的安全吗？
A: 是的。数据在本地用你的密码加密后才上传到 GitHub，GitHub 只能看到加密后的乱码。

### Q: 支持多人使用吗？
A: 目前设计为双人使用。如需更多人，可以复制代码修改。

### Q: 数据会丢失吗？
A: 不会。数据同时存储在：
1. 浏览器 LocalStorage（本地）
2. GitHub Gist（云端备份）
3. 可随时导出 JSON 备份

### Q: 如何迁移到新设备？
A: 在新设备打开应用，输入相同的 GitHub Token 和加密密码，数据会自动同步。

## 🐛 故障排除

### 无法同步数据
1. 检查 GitHub Token 是否正确
2. 确认 Token 有 `gist` 权限
3. 检查网络连接

### 日历显示异常
1. 清除浏览器缓存
2. 检查系统日期时间是否正确
3. 刷新页面

### 图表不显示
1. 检查是否已记录至少两天的数据
2. 确认 Chart.js 已正确加载
3. 查看浏览器控制台错误信息
